# 🏃‍♂️ Инструкция по запуску проекта "Velocity Sport" в университете

Этот проект разработан на фреймворке **Laravel 8** с использованием легкой базы данных **SQLite**. 
Главное преимущество этой сборки — **база данных уже встроена внутрь проекта** (файл `database/database.sqlite`). Вам **не нужно** устанавливать и настраивать локальный сервер MySQL в университете!

---

## 💾 Шаг 1. Что взять с собой на флешке
Скопируйте всю папку `Praktika` на флешку **полностью**.
> [!IMPORTANT]
> **Обязательно сохраните папку `vendor`!** 
> На компьютерах в университете может не быть установленного пакетного менеджера Composer или доступа в Интернет. Скопировав папку `vendor` целиком, вы избавляете себя от необходимости скачивать зависимости на месте — проект запустится "из коробки".

---

## 💻 Шаг 2. Подготовка на компьютере в университете
1. Вставьте флешку в компьютер.
2. **Обязательно скопируйте** папку `Praktika` с флешки на локальный диск компьютера (например, на **Рабочий стол** или диск `D:`).
   > [!WARNING]
   > Не запускайте проект прямо с флешки! Работа с файлом базы данных SQLite по USB-соединению может вызывать зависания интерфейса или ошибки доступа операционной системы.

---

## 🚀 Шаг 3. Способы запуска сайта

### Вариант А. Через встроенный сервер PHP (Рекомендуемый и самый быстрый!)
Этот способ работает на любом компьютере, где установлен PHP версии 8.0 или выше.

1. Откройте командную строку в папке с проектом:
   - Зайдите в скопированную папку `Praktika`.
   - В адресной строке проводника (сверху, где написан путь к папке) сотрите всё, напишите `cmd` и нажмите **Enter**. Откроется черное окно командной строки прямо в нужной директории.
2. Проверьте доступность PHP, введя команду:
   ```bash
   php -v
   ```
   *Если версия PHP 8.0 или выше — отлично!*
3. Запустите сервер Laravel:
   ```bash
   php artisan serve
   ```
4. Если консоль выдала `Starting Laravel development server: http://127.0.0.1:8000`, откройте браузер и перейдите по адресу:
   👉 **[http://127.0.0.1:8000](http://127.0.0.1:8000)**

---

### Вариант Б. Если команда `php` не найдена в системе
Часто на компьютерах установлен **OpenServer**, но путь к PHP не прописан в системных переменных. В этом случае мы запустим PHP напрямую из OpenServer.

1. Найдите, куда установлен OpenServer (обычно это `C:\OpenServer`, `D:\OpenServer` или `C:\OSPanel`).
2. Запустите командную строку (`cmd`) в папке вашего проекта `Praktika` (как описано в Варианте А).
3. Запустите сервер, указав прямой путь к исполняемому файлу PHP внутри OpenServer. 
   Например, если OpenServer установлен на диске `D:`:
   ```bash
   D:\OpenServer\modules\php\PHP_8.0\php.exe artisan serve
   ```
   *(Если OpenServer установлен на диске `C:`, замените букву диска: `C:\OpenServer\modules\php\PHP_8.0\php.exe artisan serve`)*
4. Откройте браузер по адресу: **`http://127.0.0.1:8000`**

---

### Вариант В. Запуск полноценного домена через интерфейс OpenServer
Если преподаватель требует запуск именно через панель OpenServer:

1. Скопируйте папку с проектом `Praktika` в директорию доменов OpenServer: `...\OpenServer\domains\`.
2. Запустите OpenServer.
3. Откройте настройки OpenServer (флажок в трее -> Настройки):
   - Перейдите во вкладку **«Модули»** и убедитесь, что в поле **PHP** выбрана версия **PHP 8.0** или **PHP 8.1** (или выше).
   - Перейдите во вкладку **«Домены»**.
   - В поле **Имя домена** напишите: `sportshop.local`
   - В поле **Папка домена** укажите путь до папки **`public`** внутри вашего проекта: `\Praktika\public` (это критически важно для безопасности и правильной работы Laravel!).
   - Нажмите **«Добавить»**, затем **«Сохранить»**.
4. Перезапустите OpenServer.
5. Откройте браузер и перейдите по адресу: **`http://sportshop.local`**

## 🛠️ Возможные проблемы и их решение

### ❌ Ошибка: `Driver [sqlite] not supported` или `could not find driver`
Это означает, что в используемой версии PHP отключена поддержка базы данных SQLite.

**Решение:**
1. Найдите файл конфигурации `php.ini` используемого PHP (в OpenServer он лежит по пути: `...\OpenServer\modules\php\PHP_8.0\php.ini`).
2. Откройте этот файл в Блокноте.
3. Нажмите `Ctrl + F` и найдите строку: `;extension=pdo_sqlite`
4. **Уберите точку с запятой `;` в начале строки**, чтобы включить расширение:
   ```ini
   extension=pdo_sqlite
   extension=sqlite3
   ```
5. Сохраните файл `php.ini` и перезапустите консоль / перезапустите OpenServer.
